## File Descriptions

### Root Level

- **beaver_choice_multi_agent.py**: Main Python implementation containing all agents, tools, and orchestration logic
- **requirements.txt**: Python package dependencies (pandas, smolagents, etc.)
- **.env.example**: Template for environment variables (API keys)
- **.gitignore**: Protects sensitive files (.env, .db) from being committed
- **README.md**: Project overview and quick start guide

### docs/ - Documentation

All project documentation and visual materials:

- **PROJECT_REPORT.md**: Comprehensive documentation covering:
  - System architecture
  - Agent responsibilities
  - Implementation details
  - Testing methodology
  - Strengths and improvements

- **agent_workflow_diagram.png**: Visual representation of the 4-agent system showing:
  - Orchestrator coordination
  - Specialist agents (Inventory, Quoting, Sales)
  - Tool associations
  - Data flow

- **agent_workflow_diagram.md**: Detailed textual description of the workflow

### data/ - Data Files

All CSV data files used by the system:

- **quote_requests.csv**: Complete dataset of customer requests
- **quote_requests_sample.csv**: Sample dataset for testing (19 requests)
- **quotes.csv**: Historical quote data for price analysis

### results/ - Test Results

Output files generated by the system:

- **test_results.csv**: Evaluation results containing:
  - Request ID and date
  - Customer details
  - Financial state (cash, inventory)
  - Agent responses

## Protected Files (Not in Git)

These files are excluded via `.gitignore`:

- **.env**: Contains sensitive API keys
- **munder_difflin.db**: SQLite database (generated at runtime)
- **__pycache__/**: Python cache files

## Usage

### Running the System

```bash
# Install dependencies
pip install -r requirements.txt
pip install smolagents litellm

# Configure environment
cp .env.example .env
# Edit .env with your API key

# Run the system
python beaver_choice_multi_agent.py
```

### Output

The system will:
1. Initialize the database
2. Process all requests in `data/quote_requests_sample.csv`
3. Generate `results/test_results.csv` with evaluation data
4. Display financial reports in console
